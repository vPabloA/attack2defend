# Pre-production Deployment on Debian

This runbook deploys Attack2Defend as a static-first navigator with a scheduled local knowledge-builder job.

The browser runtime does **not** call public APIs. Public APIs are called only by the scheduled builder. The UI consumes the local bundle generated by the builder.

---

## 1. System packages

```bash
sudo apt update
sudo apt install -y git python3 python3-venv python3-pip nodejs npm nginx cron
```

Recommended Node.js version: 20 LTS. If Debian packages provide an older Node.js version, install Node.js 20 from NodeSource or your approved internal package repository.

---

## 2. Clone repository

```bash
sudo mkdir -p /opt/attack2defend
sudo chown "$USER":"$USER" /opt/attack2defend
git clone https://github.com/vPabloA/attack2defend.git /opt/attack2defend
cd /opt/attack2defend
```

---

## 3. Python environment

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e '.[dev]'
```

---

## 4. First knowledge-base bootstrap

### Fast curated validation

Use this first to validate the local package quickly:

```bash
cd /opt/attack2defend
. .venv/bin/activate
python scripts/knowledge_builder/build_knowledge_base.py
python scripts/knowledge_builder/validate_bundle.py data/knowledge-bundle.json
pytest -q
```

### Pre-production public-source bootstrap

Use this for the first pre-production bundle:

```bash
cd /opt/attack2defend
. .venv/bin/activate
bash scripts/bootstrap_preprod.sh
pytest -q
```

If `NVD_API_KEY` is available, the bootstrap script automatically enables recent NVD enrichment:

```bash
export NVD_API_KEY="<optional-api-key>"
bash scripts/bootstrap_preprod.sh
```

Expected generated files:

```text
data/nodes.json
data/edges.json
data/indexes.json
data/coverage.json
data/routes.json
data/metadata.json
data/knowledge-bundle.json
data/knowledge-bundle.last-good.json
app/navigator-ui/public/data/knowledge-bundle.json
app/navigator-ui/public/data/knowledge-bundle.last-good.json
data/raw/                         # public-source cache when enabled
data/snapshots/<timestamp>/
```

## 6. Nginx static hosting

Example Nginx site:

```nginx
server {
    listen 8080;
    server_name _;

    root /opt/attack2defend/app/navigator-ui/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /data/ {
        alias /opt/attack2defend/app/navigator-ui/public/data/;
        add_header Cache-Control "no-store, no-cache, must-revalidate, proxy-revalidate";
        expires off;
    }
}
```

Install:

```bash
sudo tee /etc/nginx/sites-available/attack2defend >/dev/null <<'EOF'
server {
    listen 8080;
    server_name _;

    root /opt/attack2defend/app/navigator-ui/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /data/ {
        alias /opt/attack2defend/app/navigator-ui/public/data/;
        add_header Cache-Control "no-store, no-cache, must-revalidate, proxy-revalidate";
        expires off;
    }
}
EOF

sudo ln -sf /etc/nginx/sites-available/attack2defend /etc/nginx/sites-enabled/attack2defend
sudo nginx -t
sudo systemctl reload nginx
```

Open:

```text
http://<server-ip>:8080
```

Validate the runtime bundle endpoint:

```bash
curl -s http://localhost:8080/data/knowledge-bundle.json | grep -E 'CVE-2021-44228|T1567|CVE-2024-37079|CWE-79|D3-MFA'
```

If this endpoint fails, the UI may fall back to the local Log4Shell development sample. Do not treat fallback mode as pre-production validation.

## 7. Cron job

Create a log directory:

```bash
sudo mkdir -p /var/log/attack2defend
sudo chown "$USER":"$USER" /var/log/attack2defend
```

Edit crontab:

```bash
crontab -e
```

Recommended pre-production cron:

```cron
# Attack2Defend public knowledge sync at 02:30
30 2 * * * cd /opt/attack2defend && . .venv/bin/activate && bash scripts/bootstrap_preprod.sh >> /var/log/attack2defend/knowledge_builder.log 2>&1
```

Curated-only fallback cron:

```cron
# Attack2Defend curated knowledge sync at 03:00
0 3 * * * cd /opt/attack2defend && . .venv/bin/activate && python scripts/knowledge_builder/build_knowledge_base.py && python scripts/knowledge_builder/validate_bundle.py data/knowledge-bundle.json >> /var/log/attack2defend/knowledge_builder.log 2>&1
```

## 8. Manual validation checklist

The UI is search-first. The first view should be blank except for search, bundle status and counters.

Search and validate these guaranteed seeds:

| Input | Expected behavior |
|---|---|
| `CVE-2021-44228` | CVE-first route renders. |
| `T1567` | ATT&CK-first route renders. |
| `CVE-2024-37079` | Modern CVE route renders. |
| `CWE-79` | CWE/AppSec route renders. |
| `D3-MFA` | D3FEND-first identity route renders. |

When public-source mode is enabled, also validate generic public nodes such as:

| Input | Expected behavior |
|---|---|
| `T1190` | ATT&CK technique from public STIX appears. |
| `CWE-79` | CWE from public CWE XML appears and keeps curated coverage. |
| `CAPEC-63` | CAPEC public node appears if present in current CAPEC XML. |

For each input, verify:

- Route Flow renders after search.
- ATT&CK Navigator tab shows layer JSON and download.
- D3FEND CAD tab shows CAD-style graph JSON and download.
- Coverage tab renders coverage/gaps when available.
- Export tab renders Markdown, route JSON, ATT&CK layer and D3FEND CAD graph.
- Clear resets query and active route.

---

## 9. Operational notes

- The UI consumes `/data/knowledge-bundle.json`.
- The builder mirrors the bundle to `app/navigator-ui/public/data/`.
- The UI has a visible fallback local sample warning for development resilience.
- AI Route Analyst runtime is intentionally pending.
- Public-source collectors are scheduled-builder capabilities, not browser runtime dependencies.
- Public-source failures are warnings by default unless `--public-fail-on-error` is used.
