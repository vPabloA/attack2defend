# Pre-production Deployment on Debian

This runbook deploys Attack2Defend as a static-first navigator with a scheduled local knowledge-builder job.

The runtime does **not** call public APIs. The UI consumes the local bundle generated by the builder.

---

## 1. System packages

```bash
sudo apt update
sudo apt install -y git python3 python3-venv python3-pip nodejs npm nginx
```

Recommended Node.js version: 20 LTS. If Debian packages provide an older Node.js version, install Node.js 20 from NodeSource or your approved internal package repository.

---

## 2. Clone repository

```bash
sudo mkdir -p /opt/attack2defend
sudo chown "$USER":"$USER" /opt/attack2defend
git clone https://github.com/vPabloA/attack2defend.git /opt/attack2defend
cd /opt/attack2defend
```

---

## 3. Python environment

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e '.[dev]'
```

---

## 4. Build and validate knowledge bundle

```bash
cd /opt/attack2defend
. .venv/bin/activate
python scripts/knowledge_builder/build_knowledge_base.py
python scripts/knowledge_builder/validate_bundle.py data/knowledge-bundle.json
pytest -q
```

Expected generated files:

```text
data/nodes.json
data/edges.json
data/indexes.json
data/coverage.json
data/routes.json
data/metadata.json
data/knowledge-bundle.json
app/navigator-ui/public/data/knowledge-bundle.json
data/snapshots/<timestamp>/
```

---

## 5. Build UI

```bash
cd /opt/attack2defend/app/navigator-ui
npm install
npm run build
```

The static build is produced under:

```text
/opt/attack2defend/app/navigator-ui/dist
```

---

## 6. Nginx static hosting

Example Nginx site:

```nginx
server {
    listen 8080;
    server_name _;

    root /opt/attack2defend/app/navigator-ui/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /data/ {
        alias /opt/attack2defend/app/navigator-ui/public/data/;
        add_header Cache-Control "no-cache";
    }
}
```

Install:

```bash
sudo tee /etc/nginx/sites-available/attack2defend >/dev/null <<'EOF'
server {
    listen 8080;
    server_name _;

    root /opt/attack2defend/app/navigator-ui/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /data/ {
        alias /opt/attack2defend/app/navigator-ui/public/data/;
        add_header Cache-Control "no-cache";
    }
}
EOF

sudo ln -sf /etc/nginx/sites-available/attack2defend /etc/nginx/sites-enabled/attack2defend
sudo nginx -t
sudo systemctl reload nginx
```

Open:

```text
http://<server-ip>:8080
```

---

## 7. Cron job

Create a log directory:

```bash
sudo mkdir -p /var/log/attack2defend
sudo chown "$USER":"$USER" /var/log/attack2defend
```

Edit crontab:

```bash
crontab -e
```

Add:

```cron
# Attack2Defend local knowledge sync
30 2 * * * cd /opt/attack2defend && . .venv/bin/activate && python scripts/knowledge_builder/build_knowledge_base.py && python scripts/knowledge_builder/validate_bundle.py data/knowledge-bundle.json >> /var/log/attack2defend/knowledge_builder.log 2>&1
```

## 8. Manual validation checklist

Search and validate these seeds in the UI:

| Input | Expected behavior |
|---|---|
| `CVE-2021-44228` | CVE-first route renders. |
| `T1567` | ATT&CK-first route renders. |
| `CVE-2024-37079` | Modern CVE route renders. |
| `CWE-79` | CWE/AppSec route renders. |
| `D3-MFA` | D3FEND-first identity route renders. |

For each input, verify:

- Route tab renders nodes.
- Actions tab renders deterministic actions.
- Graph tab renders route graph.
- MITRE Views tab renders links/layer export.
- Coverage tab renders coverage/gaps when available.
- Export tab renders Markdown/JSON/Navigator layer.

---

## 9. Operational notes

- The UI consumes `/data/knowledge-bundle.json`.
- The builder mirrors the bundle to `app/navigator-ui/public/data/`.
- The UI has a fallback local sample for development resilience.
- AI Route Analyst runtime is intentionally pending.
- Public source collectors are future work; current MVP uses curated seeds.
